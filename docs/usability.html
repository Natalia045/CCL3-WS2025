Hypotheses / Testable Questions

Hypothesis 1
If users are asked to remove an item, then they will fail to find a solution, because there is no visible "Delete" or "Swipe to remove" affordance.

  • IV: Presence of a visible delete button
  • DV: User success rate in removing an item

Hypothesis 2
If the "X" button is on the right, then users will click the top-left area first, because they expect a standard Android "Back" navigation.

  • IV: Placement of the close action
  • DV: Location of user click

Hypothesis 3
If the visual progress ring remains at a static "100% full" and "Red" state for all values above the 50g limit, then users will experience a loss of Relative Impact Sense, because the UI no longer provides a visual delta between a minor overstep (e.g., 55g) and a major health risk (e.g., 120g).

  • IV: Progress ring behavior above the limit
  • DV: Risk differentiation

Test Method

  • 3 task-based tests derived from the hypotheses
  • Google Form questionnaire
  • Single Ease Question (SEQ)

User Test Tasks

Task 1:
Make a food entry to today's intake and then delete it.

Task 2:
Go to the Add-Page again, then navigate back to the Home-Page.

User Test Results

Completed on January 29, 2026.

Participants

We conducted usability testing with 6 participants to evaluate the app's core functionality and navigation. The group consisted of 4 males and 2 females with an average age of 22.7 years.

Quantitative Results

Task Success: All participants (100%) successfully completed the core tasks (Entry, Delete, Navigation, Exceed Limit), with only one instance of assistance required.

Psychometric Scoring: Users rated the tasks as highly intuitive. The Single Ease Question (SEQ) scores averaged between 6.5 and 7.0 (out of 7), and the System Usability Scale (SUS) score exceeded 95, placing the app in the "Excellent" percentile.

Qualitative Feedback

Participants praised the application for being "very easy" to navigate and universally preferred the "X" button over gesture controls. However, several areas for improvement were identified:

  • Challenge Feedback: Users felt finishing a challenge lacked excitement and requested more visual rewards (colors, popups).
  • Data Entry: Users were confused that the "Add" button was unresponsive when the input value was "0".
  • Controls: Some users found the slider difficult for precise inputs and found the "Quick Add" feature confusing in its current location.

Design Changes Based on Feedback

Based on these observations, the following updates were implemented:

  • Visual Feedback: The progress ring now turns orange as users approach their limit, and completed challenges turn purple.
  • Bug Fixes: Fixed the issue preventing "0" value entries and corrected the "X" button visibility in Dark Mode.
  • Layout Refinement: Swapped the "Quick Add" and "Search" positions and made the Edit screen scrollable for better accessibility.
